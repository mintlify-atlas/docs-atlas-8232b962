## Overview

ZeroStarter is actively evolving with planned integrations for AI, analytics, background tasks, email, payments, and more. This roadmap outlines upcoming features and their implementation priorities.

<Warning>
  **Important for AI Assistants**: The following features are NOT currently implemented unless marked as "Implemented". Do not assume availability of these tools.
</Warning>

## Feature Details

### AI Integration

<AccordionGroup>
  <Accordion title="Vercel AI SDK" icon="brain">
    **Status:** Planned | **Priority:** High
    
    Integration with [Vercel AI SDK](https://ai-sdk.dev) to enable AI-powered features.
    
    **Use Cases:**
    - AI-powered customer support chatbots
    - Content generation and summarization
    - Intelligent search and recommendations
    - Natural language processing for user inputs
    
    **Implementation:**
    - Integration with OpenAI, Anthropic, and other AI providers
    - Type-safe AI function calling
    - Streaming responses support
    - Server-side and client-side AI hooks
  </Accordion>
</AccordionGroup>

### Background Tasks

<AccordionGroup>
  <Accordion title="Inngest" icon="server">
    **Status:** Planned | **Priority:** Medium
    
    Integration with [Inngest](https://www.inngest.com) for reliable background job processing.
    
    **Use Cases:**
    - Email sending and notifications
    - Data processing and ETL jobs
    - Scheduled reports and analytics
    - Webhook processing
    - Retry logic for failed operations
    
    **Implementation:**
    - Type-safe function definitions
    - Built-in retry and error handling
    - Visual workflow builder
    - Integration with Hono API routes
  </Accordion>
  
  <Accordion title="Trigger.dev" icon="rocket">
    **Status:** Planned | **Priority:** Medium
    
    Alternative integration with [Trigger.dev](https://trigger.dev) for background jobs.
    
    **Use Cases:**
    - Long-running background processes
    - Scheduled cron jobs
    - Multi-step workflows
    - Integration with external APIs
    - Task queuing and processing
    
    **Implementation:**
    - TypeScript-first approach
    - Visual workflow editor
    - Built-in observability
    - Integration with existing API structure
  </Accordion>
</AccordionGroup>

### Internationalization

<AccordionGroup>
  <Accordion title="next-intl" icon="language">
    **Status:** Planned | **Priority:** Medium
    
    Integration with [next-intl](https://next-intl.dev) for Next.js-optimized i18n.
    
    **Use Cases:**
    - Next.js App Router integration
    - Type-safe translations
    - Server and client components support
    - Locale-based routing
    - SEO-friendly URLs
    
    **Implementation:**
    - Built for Next.js 13+ App Router
    - TypeScript-first approach
    - Automatic locale detection
    - Integration with Next.js middleware
  </Accordion>
  
  <Accordion title="i18next" icon="globe">
    **Status:** Planned | **Priority:** Low
    
    Integration with [i18next](https://www.i18next.com) for comprehensive i18n.
    
    **Use Cases:**
    - Multi-language support
    - Locale-based content rendering
    - Date, time, and number formatting
    - Right-to-left (RTL) language support
    - Language detection and switching
  </Accordion>
</AccordionGroup>

### Already Implemented

<AccordionGroup>
  <Accordion title="Scalar API Documentation" icon="file-lines">
    **Status:** ✅ Implemented
    
    Integration with [Scalar](https://scalar.com) for beautiful API documentation.
    
    **Features:**
    - Auto-generated API documentation at `/api/docs`
    - OpenAPI spec available at `/api/openapi.json`
    - Interactive API explorer with request testing
    - Code samples for multiple languages
    - Full request/response schema documentation
    
    See the [Type-Safe API documentation](/core-concepts/type-safe-api) for usage details.
  </Accordion>
  
  <Accordion title="PostHog Analytics" icon="chart-line">
    **Status:** ✅ Implemented
    
    Integration with [PostHog](https://posthog.com) for product analytics.
    
    **Features:**
    - Product analytics and user tracking
    - Feature flags and A/B testing
    - Session recordings and heatmaps
    - User funnels and retention analysis
    
    See the [Analytics documentation](/features/analytics) for configuration.
  </Accordion>
</AccordionGroup>
